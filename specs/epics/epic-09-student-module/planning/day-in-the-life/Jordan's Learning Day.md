### 8:00 AM - American History Lecture (90 min)
**Topic**: "Causes of the Civil War"

**Without Transcript Parser**:
- Brings laptop, tries to type notes

- Jordan reads slide, misses verbal explanation

- Loses place in notes, can't keep up

- Leaves lecture stressed - knows notes are incomplete

**With Transcript Parser (Cloud AI)**:
- Opens app, starts recording
- Settings: Dyslexia-friendly mode (OpenDyslexic font, cream background, 16pt text)
- Tags: #History #CivilWar #LectureNotes
- Focuses entirely on listening (his strength!)
- Professor mentions "Bleeding Kansas" - Jordan remembers from reading but couldn't spell it
- Taps bookmark when professor explains Dred Scott case (wants to review later)
- Relaxed - knows transcript will have correct spelling, complete content

**With Transcript Parser + Meta Ray-Ban Glasses (Hands-Free)**:
- Wears Meta Ray-Ban Smart Glasses
- **Dyslexia benefit**: No typing, no reading during lecture - pure listening (his strength!)
- Starts recording with voice command (no buttons to press)
- **Hands-free note-taking**: Can sketch timelines, maps in notebook without managing devices
- **Reduced cognitive load**: Not juggling laptop/phone/notes simultaneously
- **Focus**: 100% attention on professor's verbal explanation (visual text is hard for him)
- After lecture: Has perfect transcript (all names spelled correctly) + his visual timeline sketches
- **Accessibility win**: Technology adapts to his learning style, not the other way around

**With Transcript Parser + Local AI (Ollama - Disability Privacy)**:

1. **Local TTS processing** (60 min):

2. **Local study guide generation** (10 min):
   - Uses Ollama (LLaMA 3 8B) to create dyslexia-friendly study guide

   - **Quality**: Tuned for dyslexia-friendly output (shorter paragraphs, simpler words)

3. **Why Jordan uses local AI**:
   - **Disability privacy**: ADA protects medical privacy - doesn't want cloud logs of TTS usage
   - **Stigma avoidance**: Doesn't want data showing he needs reading support
   - **Offline access**: Study anywhere, anytime (no internet dependency)
   - **Unlimited usage**: No cloud API limits or costs for heavy TTS use
   - **Trust**: Learning disability data stays on his device

4. **Independence**:

## üí° Key Insights

### What Made the Difference
1. **Leverages auditory strengths** - Listening comprehension is Jordan's superpower
2. **Reduces reading load** - Audio + simple text + TTS = multi-modal input
3. **Perfect spelling in transcripts** - Eliminates dyslexia-related errors
4. **Dyslexia-friendly formatting** - OpenDyslexic font, high contrast, clear structure
5. **Organized study guides** - Reduces cognitive load of information processing
6. **Text-to-speech integration** - Can listen to transcripts, not just read
7. **Spell-tolerant search** - Finds "Missouri Compromise" even if he types "Missuri"

### Feature Usage
- üìù **Transcription**: Every lecture (100%)
- üîä **Text-to-speech**: Daily, primary mode (80% TTS, 20% reading)
- üé® **Dyslexia-friendly fonts**: Always enabled
- üìä **AI study guides**: All major topics
- üóÇÔ∏è **Flashcards**: 100+ cards/week (all AI-generated)
- üîç **Spell-tolerant search**: 10+ searches/day
- üì± **Mobile**: 40% (flashcard review, quick lookups)
- üéß **Audio + text sync**: When studying (highlights text as audio plays)
- üîí **Local AI (Ollama)**: Used for all TTS and study guides (100% - disability privacy)
- üëì **Meta Ray-Ban Glasses**: Used for hands-free, distraction-free lecture capture (dyslexia accommodation)

### Success Factors for Dyslexic Students
- **Multi-modal input** - See + hear + interact (strongest combo)
- **Reduce reading dependency** - Audio-first approach
- **Eliminate spelling barriers** - AI generates correct spelling
- **Clear structure** - Predictable layouts, visual hierarchy
- **Customizable display** - Font, size, spacing, colors
- **Built-in TTS** - No external app needed
- **Organized output** - Study guides, outlines reduce cognitive load
- **Spell-tolerant search** - Works with dyslexic spelling attempts

## üéØ Implications for Design

### Must-Have Features (Based on Jordan's Day)
1. **Dyslexia-friendly fonts** - OpenDyslexic, Comic Sans, Arial (user choice)
2. **High contrast modes** - Cream/black, not white background
3. **Integrated text-to-speech** - Built-in, seamless
4. **Audio + text sync** - Highlight words as audio plays
5. **Large, adjustable text** - 14-18pt default, easily customizable
6. **Spell-tolerant search** - Fuzzy matching for common dyslexic patterns
7. **AI-generated study guides** - Clear structure, visual hierarchy
8. **Perfect transcript spelling** - Eliminates primary dyslexia barrier
9. **Minimal clutter** - Clean, simple UI
10. **Keyboard navigation** - Full functionality without mouse

### User Flow Priorities
1. Record lecture ‚Üí Listen (not take notes) ‚Üí Get transcript with correct spelling
2. Search topic (tolerates misspelling) ‚Üí Find content ‚Üí Listen with TTS
3. Generate study guide ‚Üí Listen to it ‚Üí Create flashcards (< 5 min total)

### Accessibility Standards
- **WCAG AAA compliance** - Highest standard
- **Screen reader compatible** - For TTS software
- **Keyboard navigation** - No mouse required
- **Customizable display** - All visual elements adjustable
- **Multi-modal alternatives** - Audio + text + visual for all content

### Metrics to Track
- **TTS usage rate** - Expect 70-80% for dyslexic users
- **Font customization** - Track which fonts are popular
- **Search error tolerance** - Measure success rate with misspellings
- **Study time reduction** - Target: 50-70% reduction
- **Cognitive load indicators** - Session duration, break patterns
- **Independence gains** - Reduction in note-taker usage

## üß† Dyslexia-Specific Design Principles

### 1. Reduce Reading Load
- Audio-first design
- Summaries, not full text
- TTS for everything

### 2. Eliminate Spelling Barriers
- AI ensures correct spelling in transcripts
- Spell-tolerant search
- Voice input for searches

### 3. Multi-Sensory Input
- See + hear + interact
- Audio + text sync
- Visual cues (icons, colors)

### 4. Clear Structure
- Predictable layouts
- Visual hierarchy (headings, spacing)
- Chunked information

### 5. Customization
- Font choice (OpenDyslexic, Comic Sans, etc.)
- Text size (14-18pt)
- Background color (cream, light blue, not white)
- Line spacing (1.5-2x)

### 6. Error Tolerance
- Spell-check with learning
- Undo/redo everything
- Fuzzy search
- Voice input alternatives
